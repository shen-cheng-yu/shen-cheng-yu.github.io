有server_number台后端服务器，每个IP地址想要命中一台服务器。共有total_num个IP地址等待去命中。理想的情况，是这些IP地址能够均匀分布于这些服务器上。offset_rate是偏离率，某台服务器的被命中次数与所有服务器的平均值如果大于这个比例，则认为这台服务器的命中情况不好。

下面给出两种哈希函数

利用IP地址的前三段

将IP的前三段作为参数加入hash函数。这样做的目的是保证ip地址前三位相同的用户经过hash计算将分配到相同的后端server。因为ip地址前三位相同通常意味着来着同一个局域网或者相邻区域，使用相同的后端服务让nginx在一定程度上更具有一致性。

目标地址散列调度

针对目标IP地址的负载均衡，是一种静态映射算法，通过一个散列（Hash）函数将一个目标IP地址映射到一台服务器。

value = hash(key)

目标地址散列调度算法先根据请求的目标IP地址，作为散列键（Hash Key）从静态分配的散列表找出对应的服务器，若该服务器是可用的且未超载，将请求发送到该服务器，否则返回空。

假设有一组服务器S = {S0, S1, ..., Sn-1}，W(Si)表示服务器Si的权值，
C(Si)表示服务器Si的当前连接数。ServerNode[]是一个有256个桶（Bucket）的
Hash表，一般来说服务器的数目会运小于256，当然表的大小也是可以调整的。

算法的初始化是将所有服务器顺序、循环地放置到ServerNode表中。若服务器的
连接数目大于2倍的权值，则表示服务器已超载。

素数乘法Hash函数，通过乘以素数使得散列键值尽可能地达到较均匀的分布：

2654435761UL是2到2^32 (4294967296)间接近于黄金分割的素数，2654435761 / 4294967296 = 0.618033987
