执行quick.sh

执行这个文件的时候会报错，形如：

修改quick.sh

查看该文件，发现COMPILE_MODULE这个参数写错了。应将其修改为：COMPILE_MODULE=("J1800_igb")。

添加库文件J1800_igb

首先还是将SmartSecuri-IoT和SmartSecuri-Platform这两个文件给放到/opt/iot目录下。
在/opt/iot/Smartsecuri-Platform/trunk/adaptor/f-stack/dpdk目录下，少了J1800_igb这个库文件，拷贝过来。然后再执行quick.sh。

添加权限

/bin/sh: /opt/iot/SmartSecuri-Platform/trunk/opensource/f-stack/dpdk/buildtools/check-experimental-syms.sh: 权限不够执行的时候，出现了这一句报错，索性就给/iot目录下的所有文件对所有用户都添加最多的权限，所以要用到-r 递归赋予权限的选项。
[root@SmartSPG6300 opt]# chmod 777 -R iot/。

部署安装

以上三步之后，quick.sh终于能够正常执行了。然后在相同的目录下，执行

首先是杀死这个进程，然后是拷贝文件，再然后是运行start_fwd.sh，后面的J1800_igb是运行时的参数，即$1位置参数。出现Successfully Start Fwd, Start Mode:mr即表示整个编译环境的搭建的工作已经完成了。

环境搭建完成的意思应该就是说，这个时候就可以用Makefile来编译连接整个项目文件了。

搭建编译环境

在设备里重新拷贝了一份不是从svn里下载的代码，然后重新部署编译环境。

  1. 修改quick.sh
  2. 将J1800的库拷贝到adaptor。这个时候要注意，库放的位置一定要正确，执行quick.sh的时候，需要在adaptor里面找到J1800，将adaptor里面的东西拷贝到opensource，之后是嵌套式的调用各个makefile，编译dpdk下面的所有模块。编译完成之后，在quick.sh的同级目录下的fwd目录下，生成release目录，里面有待会执行cp命令时所需的J1800_igb文件。
  3. 执行命令运行。

之所以已经在adaptor里面存放了J1800，但还是会Fail Compile x86_64-native-linuxapp-gcc_J1800_igb，是因为目录放的不对。ls J1800之后看到的还是J1800文件夹，再次ls之后才看到J1800下面的几个子文件夹。这就是说，J1800放置的目录深了一层。向上提取一层，之后执行quick就可以找到J1800了。

使用编译机编译

登录

编译机IP192.168.2.222，密码smartsecuri。

建立代码文件夹

建立/home/iot/shenchengyu目录，在该目录下存放代码SmartSecuri-Platform和SmartSecuri-IoT。

为什么使用编译机

从SVN上下载的代码，直接放到设备上，是加密的，无法编译。用vim cat查看是乱码。

使用专用的编译设备，将svn下载的代码放到编译机上面，也是无法直接查看，但是可以直接编译。

编译流程

在编译机上面编译。/home/iot/shenchengyu/Smartsecuri-IoT/trunk/gateway/build目录下，quick.sh编译。

将编译机里面的fwd/release/J1800_igb/fwd_J1800_igb 复制到自己设备的/SPG/bin/fwd/pkg/J1800_igb/文件夹下面，同名文件替换。

将自己设备上的/SPG/bin/startup.sh和/SPG/bin/fwd/start_fwd.sh文件用外部文件替换掉。

自己的设备重启。

在自己的设备上，pkill fwd，/SPG/bin/fwd/start_fwd.sh J1800_igb进行编译。
