挂载

Linux 系统中“一切皆文件”，所有文件都放置在以根目录为树根的树形目录结构中。在 Linux 看来，任何硬件设备也都是文件，它们各有自己的一套文件系统（文件目录结构）。

因此产生的问题是，当在 Linux 系统中使用这些硬件设备时，只有将Linux本身的文件目录与硬件设备的文件目录合二为一，硬件设备才能为我们所用。合二为一的过程称为“挂载”。

挂载，指的就是将设备文件中的顶级目录连接到 Linux 根目录下的某一目录（最好是空目录），访问此目录就等同于访问设备文件。

注意：并不是根目录下任何一个目录都可以作为挂载点，由于挂载操作会使得原有目录中文件被隐藏，因此根目录以及系统原有目录都不要作为挂载点，会造成系统异常甚至崩溃，挂载点最好是新建的空目录。可以在挂载之前新建一个目录用来做挂载点。

举个例子，我们想通过命令行访问某个 U 盘中的数据，图示为 U 盘文件目录结构和 Linux 系统中的文件目录结构。


可以看到，U 盘文件系统已经成为 Linux 文件系统目录的一部分，此时访问 /sdb-u/ 就等同于访问 U 盘。

什么是挂载，Linux挂载详解

fdisk 分区

fdisk -l 查看分区

Device：分区的设备文件名。
Boot：是否为启动引导分区，在这里 /dev/sda2 为启动引导分区。
Start：起始柱面，代表分区从哪里开始。
End：终止柱面，代表分区到哪里结束。
Blocks：分区的大小，单位是 KB。
id：分区内文件系统的 ID。
System：分区内安装的系统是什么。

U 盘的设备文件名是在插入 U 盘后系统自动分配的。查看分区的时候，可以看到U盘的设备文件名是什么。

fdisk交互

使用fdisk -m获取帮助，帮助里列出了 fdisk 可以识别的交互命令。

符号 交互命令
d 删除一个分区
m 查看帮助
n 新建一个分区
p 显示分区列表
q 不保存退出
w 保存退出
x 附加功能

使用fdisk /dev/sdb对该硬盘进行分区。

在linux系统下进行磁盘分区的时候，默认的分区类型就是Linux。

No free sectors available这是因为现有的分区已经将所有的空间都用完了，没有可用的扇区了。

exFAT

exFAT（Extended File Allocation Table File System，也称作FAT64，即扩展文件分配表）是Microsoft在Windows Embeded 5.0中引入的一种适合于闪存的文件系统，为了解决FAT32等不支持4G及其更大的文件而推出。对于闪存，NTFS文件系统不适合使用，因为NTFS是日志文件系统，需要记录详细的读写操作，需要不断读写，比较伤闪盘芯片，exFAT更为适用。

但是设备的CentOS系统不直接支持这个格式，需要转换成FAT32才可以直接进行挂载。
CentOS只支持FAT32和vfat两种格式的。

编码格式是保存在一个环境变量中，此时是GB10830。

如果挂载后，打开发现是乱码，这时候要指定编码格式进行挂载

-t是指定欲挂载的文件系统类型。

Partition table entries are not in disk order

现在分区表中分区的顺序与硬盘的物理顺序不一致。
fdisk /dev/sdb对硬盘进行分区，先后输入x f w即可。

相关的查看命令

使用mount -l查看当前已经挂载的设备：

使用mount -a命令检查有无疏漏挂载的文件：
-a 选项的含义是自动检查 /etc/fstab 文件中有无疏漏被挂载的设备文件，如果有，则进行自动挂载操作。/etc/fstab 文件是自动挂载文件，系统开机时会主动读取 /etc/fstab这个文件中的内容，根据该文件的配置，系统会自动挂载指定设备。

卸载

umount /dev/sdb使用设备名进行卸载。
卸载的时候，如果提示设备忙，是因为此时正在挂载目录下，卸载之前需要先退出挂载目录。

卸载之后，挂载目录下隐藏的文件会再次显现出来。
