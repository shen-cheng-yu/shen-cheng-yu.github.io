生成ssh密钥

输入ssh-keygen后，在指定文件id_rsa.pub下面得到密钥，输入到github设置里面去保存。这是为了将==git==和==github==建立连接，设置里面保存的是一个十六进制的数据，我的是74:d1:16:d5:1a:df:d3:30:44:d9:83:9d:aa:81:a5:b7，
这里会得到一个密钥数据的randomart image，这好像是根据这个十六进制数据进行MD5加密后得到的一个类似于二维码的图形。具体加密原理，下面这个链接的作者介绍的非常好。

pull与push

各行代码意义简介：

  1. 初始化本地文件夹，之后在要上传的文件夹里面会多出来一个.git文件夹

  2. 将本地与仓库连接起来。这里涉及到https和ssh两种安全协议。区别可以参考链接。
    https与ssh的区别
    第一次用的是https，确实弹出了用户名和密码的登陆窗口，如下图。但是之后用https并没有再弹出登陆窗口。整个过程搞得迷迷糊糊的。并且，登陆时的用户名输入的是邮箱，输入xtncsg是没有用的！

  3. 将要上传的文件保存到缓存区。这里如果要上传全部文件，就输入add .

  4. 推到master里面去，这里可以写上注释
  5. 推到远程仓库里面去
  6. 很多情况下，会出现各种各样的错误，这一句是强制推进去，不过这样远程仓库原有的内容都会丢失。慎用！！！
