括号

$((exp))用于计算整数表达式和$[]的效果是一样的。$((25+2))结果是27。

大括号有一种特殊用法，{1..10}相当于从1到10，touch {a..d}.txt 结果为a.txt b.txt c.txt d.txt。大括号中没有空格。

命令组

( echo "astr !=bstr";echo "hello" )小括号里可以放好几个命令，彼此之间用分号隔开。括号中的命令将会新开一个子shell顺序执行，所以括号中的变量不能够被脚本余下的部分使用 。

{ cmd1;cmd2;cmd3;} 在当前shell顺序执行命令cmd1,cmd2,cmd3, 各命令之间用分号隔开，最后一个命令也必须带分号。

dirname

dirname命令可以取给定路径的目录部分。这个命令很一般用在shell脚本中，用于取得脚本文件所在目录，然后将当前目录切换过去。Linux下还有一个命令是basename，它与dirname相反，是取得文件名称部分。

linux下dirname $0获取当前脚本执行的路径名

引号

单引号屏蔽各种特殊字符。
双引号支持内部字符有特殊含义,比如换行，变量，传递的参数等。
反引号中字符串作为shell命令执行，并返回执行结果。$()等价于` `

linux如果echo要让转义字符发生作用，就要使用-e选项,且转义字符要使用双引号。

-e激活转义字符。使用-e选项时，若字符串中出现以下字符，则特别加以处理，而不会将它当成一般文字输出：

字符 含义
\a 发出警告声
\b 删除前一个字符
\c 最后不加上换行符号
\f 换行但光标仍旧停留在原来的位置
\n 换行且光标移至行首

export

程序输出：

脚本中export的变量在启动其他脚本的时候会复制一份传入其他脚本。其他脚本中对此变量的修改并不会在离开脚本后生效。

使用场景：

在脚本A中定义了一个变量V赋值为1，export了变量V，在脚本A中启动脚本B，在脚本B中变量V的值就是1，如果在脚本B中修改了V的值为2，那么脚本B结束后，脚本A中的V的值依然是1。

脚本A中export的变量在启动其他脚本的时候会复制一份传入其他脚本，传入的值是调用其他脚本>的时的值。并不是export时的值。 在脚本A中定义了一个变量V赋值为1，export了变量V，然后修改V为2，在脚本A中启动脚本B，在脚本B中变量V的值就是2。

1.sh中只有var2得到了export，所以在2.sh中输出var1的值，将会得不到任何值。

连续执行多条命令

&&和||这两个符号连接命令时，意思是只有当前一个命令执行成功或者执行失败时，后面一个命令才会继续执行。

当使用;来连接时，则不论前面的命令执行是否成功，后面的命令都会继续执行。

这种成功或者失败的关系是从左向右顺序传递的，遇到不执行的则跳过。
以最后一条命令为例，echoo 1执行失败， 所以echo 2执行。因为echo 2执行成功，所以echo 3不执行。 因为echo 2执行成功，所以echo 4不执行。 因为echo 2执行成功，所以echoo 5执行。因为echoo 5执行失败，所以echo 6不执行。 最后echo 7执行了。

指定样式输出文字

格式：echo -e "\033[背景颜色;字体颜色;显示效果m待输出的字符串\033[0m"
其中，\033[是命令的开头部分，中间可以填写的字段包括背景颜色，字体颜色，显示效果这三个参数以及待输出的字符串，\033[0m]是命令的结束部分（用以将颜色置回）。

这三个参数的可选值各不相同。背景颜色取值在40-47之间，字体颜色取值在30-37之间。

显示效果的取值如下：

值 效果

1 加深高亮显示
2 减弱低亮显示
4 下划线
5 闪烁
7 反显（替换背景色和字体颜色）
8 消隐（不在屏幕上显示输出）

awk

awk是把文件逐行的读入，以空格为默认分隔符将每行切片，切开的部分再进行各种分析处理。
基本格式： awk '{pattern action}' {filenames}
其中 pattern 表示 AWK 在数据中查找的内容，而 action 是在找到匹配内容时所执行的一系列命令。pattern就是要表示的正则表达式，用斜杠括起来。

pattern是/function/进行正则表达式匹配，匹配到了就执行{print $2}这个action，即打印出每一行第二列的元素。默认是对每一行以空格进行分隔。最后写上文件名。

exit -1

shell编程里，exit用于退出程序（退出整个shell脚本文件）。
退出码应该写在0-255的范围内。

shell应该是用8位数据来存储这个退出码，且这个退出码使用unsigned int来存储。
所以如果写exit -1，计算机中的数据都是用补码来存储。-1的补码是11111111。将这个数据用unsigned int来解析，得到的结果就是255。

同理，如果exit -109，得到的返回码就是147，猜想验证通过。

执行完这个文件之后，在终端输入echo $?来获取脚本文件执行之后的返回码，即该文件中最后一条语句执行的状态返回码，得到是147。
