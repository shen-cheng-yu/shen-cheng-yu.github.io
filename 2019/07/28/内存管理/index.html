内存分区

未初始化的全局变量（.bss段）

该段存放那些没有初始化或者初始化为0的全局变量。

执行ls -hl a.out的结果是-rwxr-xr-x 1 root root 8.4K 7月 28 04:03 a.out。
全局数组的大小是4M，但是可执行文件的大小只有8.4K，说明bss类型的全局变量只占用运行时的内存空间，而不占用文件空间。

初始化后的全局变量（.data段）

将全局数组初始化之后，int bss_array[1024*1024] = {1};结果就变成了-rwxr-xr-x 1 root root 4.1M 7月 28 04:46 a.out。
如果数据全是0，编译器就会将他作为bss处理，所以data段是存放那些初始化为非0的全局变量。

常量数据（.rodata段）

read only data是用来存放常量数据，当然常量数据（和指令编码在一起的常量数据）也有可能存放在代码段中（text）。
字符串常量，编译器会自动去重，保证一个字符串在一个可执行文件中只存在一份复制。
rodata是在多个进程之间共享的，这样可以提高运行空间利用率。
在有些环境下，rodata段中的数据可以在运行时直接读取，不用加载到RAM中。
字符串会被编译器自动放到rodata中，其他数据需要手动加const关键字。

代码

存放代码（如函数）和部分整数常量。

栈

存放临时变量和函数参数，计算机中的栈顶地址是向低地址延伸的。

堆

内存泄漏

动态分配了一块内存之后，但是没有释放，就称为内存泄漏（Memory Leak）。

缓冲区溢出

分配了一块内存之后，读写都只能在这个分配的范围之内。读多了就会读到随机的数据，写多了就会造成随机的破坏。这种情况称为缓冲区溢出（Buffer Overflow）。

内存分配方式

静态存储区分配

内存在编译时就已经分配好，在程序的整个运行期间都存在，如全局变量，static静态变量等。这些变量在程序的整个过程中都需要用到，单独划出一块存储区保存，并且在整个运行过程中该存储区内容不清空。

在栈上创建

执行函数时，函数内部的局部变量是在栈上创建的，函数执行结束时这些变量所占的内存会自动释放。栈上分配内存效率高，但是容量有限。

在堆上创建

动态内存的生存周期由程序员自己指定。
