SSH

SSH（Secure Shell）为建立在应用层基础上的安全协议。SSH 是目前较可靠，专为远程登录会话和其他网络服务提供安全性的协议。利用 SSH 协议可以有效防止远程管理过程中的信息泄露问题。传统的网络服务程序，如：ftp、pop和telnet在本质上都是不安全的，因为它们在网络上用明文传送口令和数据，别有用心的人非常容易就可以截获这些口令和数据。而且，这些服务程序的安全验证方式也是有其弱点的， 就是很容易受到“中间人”（man-in-the-middle）这种方式的攻击。所谓“中间人”的攻击方式， 就是“中间人”冒充真正的服务器接收你传给服务器的数据，然后再冒充你把数据传给真正的服务器。服务器和你之间的数据传送被“中间人”一转手做了手脚之后，就会出现很严重的问题。使用SSH，还有一个额外的好处就是传输的数据是经过压缩的，所以可以加快传输的速度。

运行

sh ./first.sh

shell类型

echo $SHELL 查看类型
结果是/bin/bash
改变类型 # csh
只能改成这一个shell类型，这个命令是又重新启动了一个新的shell，他是最初的shell（bash）的下一级shell或者说是子shell。并且启动了之后会禁用上下键输入命令。exit退出这个shell。

输入输出重定向

sh ./first.sh>tmp.txt
将程序的输出结果重定向输入到另外一个文件里面去。

Shell变量

shell所有变量的取值都是一个字符串。

环境变量

环境变量是shell在开始执行时就已经定义了的一些和系统的工作环境相关的变量，这些变量用户还可以重新定义。

查看用户名，终端等一些环境变量的命令。

主提示符：特权用户是#，普通用户是$，每一行命令最开始的提示符。
辅助提示符：在一行命令之后输入\或者系统判断命令在这一行没有输入完，下一行继续输入时，下一行开头得到提示符就是>。

自定义变量

自己定义的变量，定义的时候不要加\$，引用的时候要加\$。
readly var=20这样子定义的变量，定义完之后就不能再修改了。

shell里面定义的变量都是这个shell程序的局部变量，只能它自己使用，不能交给其他shell程序使用。但是export var或者export var=20这样的变量就可以在其他的shell程序里面使用到。

位置参数

在调用一个shell程序的时候，程序名之后的各个参数就叫做位置参数，之间以空格分隔，会按顺序的替换掉程序中的\$1，\$2，\$3，\$4等。\$0是程序的名字，程序里面不应该出现。

ip地址

A类地址最开始是0，B类地址最开始是10，C类地址最开始是110……….
A类地址的范围是0.0.0.0~127.255.255.255，其中127.0.0.0~127.255.255.255都是单独拿出来用作保留地址的。
其中，127.0.0.1是回环地址，用来测试时使用，能ping通回环地址表示本机的TCP/IP协议安装正常。

Linux中将127.0.0.1与localhost是绑定起来的。
三类地址的私有地址是：
A类地址：10.0.0.0～10.255.255.255
B类地址：172.16.0.0～172.31.255.255
C类地址：192.168.0.0～192.168.255.255

PING通127.0.0.1代表本机TCP/IP协议正常。 不通 则表示协议没有装好。
能PING通本机IP地址代表本机网卡通信正常。不通网卡通信不正常。
能PING通网关代表计算机能够连通代理服务器或者internet ，不通则无法连接互联网。
能PING通DNS代表计算机能够通过DNS解析域名，无法打开网页。

指定ping的次数

ping 192.168.2.166 -c 3

修改设备的默认网关

之前KNI0口是1网段，而MGT口不能和KNI0口冲突，所以当时把MGT口设置成了2网段。所以设备上网的默认网关理应是192.168.2.1，但是它实际的默认网关却是KNI0口下的192.168.1.1，这个地址不具有路由转发的功能，所以设备ping不通192.168.1.1，8.8.8.8，www.baidu.com，114.114.114.114。
route -n查看设备的默认网关。
删除掉192.168.1.1这个默认网关即可。route del default gw 192.168.1.1，然后默认网关，外网ip，域名，dns服务器地址就都可以ping的通了。
