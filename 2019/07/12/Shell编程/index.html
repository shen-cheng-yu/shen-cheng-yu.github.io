运行shell文件

为此文件赋予可执行权限，如果直接执行hello_world.sh，这时会默认从$PATH环境变量中去寻找，由于我们未将此文件配置在环境变量中，因此会找不到。所以，我们从当前目录找。还可以自己指定shell来执行，这时候#!/bin/bash将不会生效。

shell类型

echo $SHELL 查看类型
结果是/bin/bash
改变类型 # csh
只能改成这一个shell类型，这个命令是又重新启动了一个新的shell，他是最初的shell（bash）的下一级shell或者说是子shell。并且启动了之后会禁用上下键输入命令。exit退出这个shell。

Shell变量

环境变量

环境变量是shell在开始执行时就已经定义了的一些和系统的工作环境相关的变量，这些变量用户还可以重新定义。
查看用户名，终端等一些环境变量的命令。

主提示符：特权用户是#，普通用户是$，每一行命令最开始的提示符。
辅助提示符：在一行命令之后输入\或者系统判断命令在这一行没有输入完，下一行继续输入时，下一行开头得到提示符就是>。

自定义变量

自己定义的变量，若是hello world这样内部含有空格的或者含有引号的，需要加上引号括起来。即hello="hello world"。注意，只是自己定义变量的时候才这样。

自己定义的变量，定义的时候不要加\$，引用的时候要加\$。
readly var=20这样子定义的变量，定义完之后就不能再修改了。

shell里面定义的变量都是这个shell程序的局部变量，只能它自己使用，不能交给其他shell程序使用。但是export var或者export var=20这样的变量就可以在其他的shell程序里面使用到。

位置参数

在调用一个shell程序的时候，程序名之后的各个参数就叫做位置参数，之间以空格分隔，会按顺序的替换掉程序中的\$1，\$2，\$3，\$4等。\$0是程序的名字，程序里面不应该出现。

获取指定位置参数的值

单纯的$0获取到的文件名是包含有路径的，所以第二行中用basename $0来获取纯净的文件名。
其中，\$var和\${var}都是取变量var的值。不过${var}能够限定变量的范围。

因此如果位置参数超过9个，那么从第十个开始就只能通过${}来获取。

通过\$*和\$@

\$* 会将命令行上提供的所有参数当作一个单词保存, 我们得到的值也就相当于是个字符串整体。一个字符串中只有一个单词。
\$@ 会将命令行上提供的所有参数当作同一字符串中的多个独立的单词。一个字符串中有很多个单词。

如果for语句中不加引号，则for默认以空格进行分割，又看不出区别。

echo ${ #hello:1:4}1是子串的起始位置，4是子串的长度。

字符串中查找字符：

echo `expr index "hello" h`hello是要查找的字符串，h是待查找的字符。这里查找到的是从0开始的序号加1的值，即第一个字符查找到的结果是1。
