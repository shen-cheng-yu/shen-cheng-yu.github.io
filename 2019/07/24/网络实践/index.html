ping

PING通127.0.0.1代表本机TCP/IP协议正常。 不通 则表示协议没有装好。
能PING通本机IP地址代表本机网卡通信正常。不通网卡通信不正常。
能PING通网关代表计算机能够连通代理服务器或者internet ，不通则无法连接互联网。
能PING通DNS代表计算机能够通过DNS解析域名，无法打开网页。

ping 192.168.2.166 -c 3 指定ping的次数
windows下使用-t选项一直ping。

nslookup反查域名

ping命令可以查看指定域名的IP地址，而nslookup命令可以反查指定IP地址的域名，不过很有可能会查不出来。

netsh命令修改IP

netsh（network shell）可以在命令行输入这条命令自动更改IP，也可以另存为.bat文件，直接运行。
name是网络连接的名字，要和网络适配器里面当前实际在用的相匹配才行。
gwmetric是默认网关的跃点数。只有在设置了默认网关的情况下才可以设置它。
source=static addr设置静态IP，source=dhcp则设置动态IP。

ifconfig命令

ether是MAC地址，网卡自出厂起，就带有一个全球唯一的MAC地址。
flags中的 UP 表示网卡处于启动的状态；BROADCAST 表示这个网卡有广播地址，可以发送广播包；MULTICAST 表示网卡可以发送多播包。

RX和TX是启动到目前为止的数据包接收与发送情况。packets是数据包数量，errors是数据包发生错误的数量，drops 是数据包丢失的数量。collisions是数据包发生冲突的情况。txqueuelen指传输数据的缓冲区的存储长度。

Ping中的TTL

如果主机与待检测主机在同一个网络内，则TTL是64，如果不在同一个网络内，则TTL默认是255。

host

host www.baidu.com命令使用DNS来获取IP地址。

tcpdump抓包

安装tcpdump yum install -y tcpdump

tcpdump抓取DNS包

XShell连接设备，开两个窗口，一个运行tcpdump -n -i any port 53命令，监听53号端口。另一个运行ping -c 3 www.baidu.com。这个命令向DNS所在的53号端口，发出将域名解析成IP地址的命令。这样第一个命令就能抓取到53号端口的数据包。
其中，-n是指不要将（包里面的）ip地址解析成域名，-i表示所要抓取的网卡名称，因为设备是用MGT口连接集线器，故可以-i MGT。

添加路由

暂时性的添加路由，重启失效。route add -net 192.168.3.0/24 gw 192.168.2.1这样设备就能通过2网段的网关到达3网段。

# route add default gw 192.168.2.1添加默认网关，del即为删除默认网关。

永久性地设置路由。/etc/sysconfig/static-routes文件中，添加any net 192.168.3.0/24 gw 192.168.2.1即可。
这个文件是network脚本执行时调用的一个文件。
