2019-07-24
按照官方文档的步骤来

  1. 修改 archer\source-src\scss\_variables.scss中的$feature-color 的颜色。
  2. 在 archer 目录下执行 npm install，然后执行 npm run build。
  3. 再重新执行 hexo g即可。

注意：修改颜色的时候，ffffff是黑色，000000才是白色。这是一个css文档，在css里面，可以对颜色的十六进制写法进行缩写。比如ff22aa可以缩写成f2a。

hexo g应该要回到blog的根目录下执行。

主要是第二步的时候，出现了很多错误。

node-sass

报错提示如下：

下载node-sass的v4.7.2时出错，C:\Users\14311\AppData\Roaming\npm-cache\node-sass\4.7.2\这个文件夹里面是空的，该下载的文件没有下载下来。

解决方案：

自己手动下载一下这个东西，可以看到C:\Users\14311\AppData\Roaming\npm-cache\node-sass\4.12.0目录下已经有文件了。不放心的话可以将这个v4.12.0的文件给v4.7.2文件夹里面也拷贝一份。
npm install 报node-sass错误

ajv@^5.0.0

报错提示：npm WARN ajv-keywords@2.1.1 requires a peer of ajv@^5.0.0 but none is installed.。
解决方案就是自己手动安装一下这个东西。

npm WARN ajv-keywords@2.1.1 requires a peer of ajv@^5.0.0 but none is installed.

fsevents

报错提示：

可以看出，这是操作系统不对应的问题。fsevent的系统应该是darwin，但是目前的系统是win32，忽略即可。

gulp build

还出现过类似这样的错误。解决方案是安装natives。

gulp运行出现gulp[27960]

run build

之后再blog的根目录下hexo g -d，这样就完成了官方文档的步骤。但是网页的背景颜色并没有变化。只是每次打开一个网页的时候，会有一个黑色的背景闪一下就没了。主要是还有一个问题没有解决，npm WARN hexo-theme-archer@1.5.0 No repository field.。

package.json

在Archer主题目录下，有一个package.json文件，可以看看。里面的dependencies是生产所依赖的包，devDependencies是开发所依赖的包。上述的npm WARN hexo-theme-archer@1.5.0 No repository field.是因为package.json里面缺少项目的仓库字段，在该文件里面加入一个"private":true的成员，以这种方式把项目声明为私有。之后再npm install就不会报这个错了。
npm warn package.json @1.0.0 no repository field

在package.json中，"prettier": "prettier --black './src/**/*.*'"这一句，我将white改为了black。

repository成员
